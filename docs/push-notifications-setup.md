# إشعارات الدفع (Push Notifications) — دليل التفعيل

التطبيق الآن يدعم نوعين من الإشعارات:

## 1) إشعار داخلي (يعمل الآن — بدون أي إعداد)
عند فتح التطبيق لأول مرة بعد إضافة محتوى جديد، يظهر تلقائياً:
- إشعار في جرس 🔔 (أعلى الصفحة)
- Toast منبثق + صوت لطيف + اهتزاز

**كيف تطلق إعلاناً جديداً مستقبلاً؟**
في `index.html`، ابحث عن مصفوفة `ANNOUNCEMENTS` وأضِف عنصراً بـ `id` فريد:
```js
{ id: 'videos-2026-07-xxx', ico:'🎬', title:'فيديوهات جديدة', sub:'...', url:'videos.html' }
```
كل إعلان يظهر **مرّة واحدة لكل مستخدم** (يُتتبّع في `localStorage` تحت `zad_announce_seen`).

---

## 2) إشعار دفع حقيقي عبر FCM (يصل والتطبيق مغلق)
البنية جاهزة بالكامل، تحتاج فقط 3 خطوات لمرّة واحدة:

### الخطوة 1: تفعيل Cloud Messaging
- في [Firebase Console](https://console.firebase.google.com) → مشروع `zadullashr`
- **Project Settings → Cloud Messaging** → فعّل الخدمة.

### الخطوة 2: مفتاح VAPID
- في نفس الصفحة → **Web Push certificates** → **Generate key pair**
- انسخ المفتاح وضعه في `js/firebase-push.js`:
```js
var VAPID_KEY = 'ضع_المفتاح_هنا';
```

### الخطوة 3: تأكد من رفع الملفات
هذه الملفات يجب أن تكون في جذر الموقع (مرفوعة بالفعل):
- `firebase-messaging-sw.js` (عامل خدمة الإشعارات في الخلفية)
- `js/firebase-push.js` (تسجيل التوكن)

> ملاحظة: الـ VAPID public key وapiKey للويب **ليست أسراراً** — آمنة في كود العميل.

---

## كيف يفعّل المستخدم الإشعارات؟
يضغط زر **«🔔 تفعيل الإشعارات»** في قائمة الجرس → يوافق على الإذن → يُسجَّل توكنه تلقائياً في
RTDB تحت `pushTokens/{uid|anon}`.

## كيف ترسل إشعاراً؟
**يدوياً (الأسهل):** Firebase Console → **Messaging → New campaign → Notifications**
- اكتب العنوان والنص، واستهدف "جميع المستخدمين" أو موضوعاً (topic).
- في **Additional options → Custom data** أضِف `url` = `videos.html` (لفتح الصفحة عند النقر).

**برمجياً (سيرفر):** استخدم FCM HTTP v1 API مع مفتاح خدمة (Service Account). أرسل إلى التوكنات
المخزّنة في `pushTokens/`. الحمولة:
```json
{
  "notification": { "title": "فيديوهات جديدة", "body": "..." },
  "data": { "url": "videos.html", "tag": "videos-update" }
}
```

## اختبار
1. افتح الموقع المنشور (HTTPS) على الموبايل، فعّل الإشعارات.
2. من Firebase Console أرسل رسالة اختبار للتوكن.
3. أغلق التطبيق → يجب أن يصل الإشعار لشريط النظام.

> القيود: الإشعار في الخلفية لا يعمل على iOS إلا إذا أضاف المستخدم التطبيق للشاشة الرئيسية (PWA مثبّتة) على iOS 16.4+.

# زاد العشر — البنية التقنية وخارطة التوسّع ومعايير الإطلاق

> مرجعٌ هندسيٌّ موجز يجمع: المكدّس التقني وميزانيات الأداء، رؤية التحوّل إلى نظامٍ بيئيٍّ إيمانيٍّ دائم (الأعمدة الأربعة)، وقائمة الفحص النهائية قبل الإطلاق.
> *ملاحظة:* الإصدار الحالي مبنيٌّ كـ **PWA (Vanilla JS + Service Worker + localStorage)**؛ الجدول أدناه يصف **المسار المستهدف (v2 Native)** كمرجعٍ معماري، مع تعليقٍ على ما هو منفَّذٌ اليوم.

---

## 1) المكدّس التقني وميزانيات الأداء

### المكدّس المستهدف (v2)
| الطبقة | التقنية المقترحة | المبرّر | الحالة اليوم (PWA) |
|---|---|---|---|
| Frontend | React Native + Expo | كودٌ مشترك iOS/Android، نشرٌ سريع (EAS) | HTML/CSS/JS responsive + PWA |
| قاعدة محلية | SQLite (expo-sqlite) | أداءٌ عالٍ، أوفلاين، حجمٌ صغير | `localStorage` (مفتاح `zad_v2`) |
| قاعدة سحابية | Supabase (PostgreSQL) | مفتوح المصدر، RLS، Realtime | غير مفعّل (موثّق في دليل الربط) |
| مزامنة لحظية | Supabase Realtime / Firebase RTDB | WebSocket، حمولة صغيرة | محوّل محلّي (LocalAdapter) |
| أسعار السوق | CoinGecko + Yahoo Finance API | مجاني/موثوق، JSON | إدخالٌ يدويٌّ أوفلاين (الزكاة/الأضحية) |
| أوقات الصلاة | Adhan-js (محلية) | تعمل أوفلاين، دقيقة | تقريبٌ زمنيٌّ + موقع المستخدم |
| التاريخ الهجري | moment-hijri / hijri-date | تصحيح ±1 يوم | محسوبٌ من مرساة عرفة |
| المصحف | Quran.com API + Local Cache | تخزينٌ بعد أول تشغيل | محتوى مُضمَّن أوفلاين |
| التحليلات | Sentry (للأخطاء فقط، مجهول) | لا تتبّع سلوكي | لا تتبّع إطلاقاً |
| المصادقة | Supabase Auth (Google/Apple OAuth) | بسيط، آمن، بلا كلمة مرور | وضعٌ مجهولٌ محلّي (UUID) |

### ميزانيات الأداء (غير قابلة للتفاوض)
- **< 3 ثوانٍ** زمن أول تحميل (حتى ظهور لوحة التحكم كاملة على 4G).
- **< 100ms** استجابة نقر المهمة (تغيّر الحالة بصرياً بلا تأخير محسوس).
- **5KB** حد أقصى لحمولة المزامنة (يضمن العمل في شبكات الحج الضعيفة).
- **< 50MB** حجم التطبيق بعد التثبيت الأول.

### استراتيجية التحميل التراكمي (Lazy Loading)
- **فوراً (Priority):** لوحة التحكم، أوقات الصلاة، قائمة المهام، التكبير.
- **عند الحاجة (Lazy):** ملفات الصوت للقرّاء، بيانات الأسعار، الفيديوهات، معجم الجذور.
- **في الخلفية (Background):** محتوى الأيام القادمة، بيانات المجموعة، بطاقات المعرفة.

> اليوم على PWA: الـ Service Worker يطبّق `network-first` مع `PRECACHE` للأصول الأساسية، وهو المكافئ العملي لاستراتيجية الأولوية/الخلفية.

## 2) خارطة التوسّع — من أداةٍ موسمية إلى نظامٍ بيئيٍّ دائم (الأعمدة الأربعة)

بعد انتهاء العشر، يتحوّل التطبيق إلى نظامٍ إيمانيٍّ دائمٍ يرتكز على أربعة أعمدة:

1. **خزانة المعرفة والمرجعية الشرعية:** محرّك بحثٍ في الأحاديث والكتب الفقهية، ملخّصاتٌ ذكية للإجابات، مكتبة فتاوى (جملتان + المصدر)، تصنيفٌ هجري، دروسٌ مصوّرة للوضوء والصلاة والأركان. *معيار:* البحث < 2 ثانية، يعمل أوفلاين.
2. **الإنتاجية الإيمانية اليومية:** تقويمٌ يوميٌّ مقسومٌ بالصلوات الخمس، جدولة المهام الدنيوية في النوافذ، نظام Streaks موسميٌّ-حرّ، رسمٌ بيانيٌّ للنمو الروحي مع نصائح. *معيار:* جدول مهامٍ بالنوافذ، تحليلاتٌ أوفلاين. **(منفَّذ جزئياً: `barnamaj.html` + اللهب + الحصاد.)**
3. **المصحف الشامل والدراسة العميقة:** معجم الجذور لكل كلمة أوفلاين، خطة ختمةٍ مرنة (30–365 يوماً)، وضع الحفظ (إخفاء الآيات)، 48 قارئاً، تفسيرٌ مختصر. *معيار:* بطاقة الجذر < 200ms، أوفلاين. **(منفَّذ جزئياً: `mushaf.html` + هدف الورد في الإعدادات.)**
4. **الأدوات الحياتية والمجتمعية:** حاسبة زكاةٍ شاملة، المساجد القريبة (GPS مُخزَّن + خرائط)، منتدى إسلاميٌّ (v3 اختياري)، أدوات السفر والحج، القبلة. *معيار:* خريطة المساجد + الزكاة أوفلاين. **(منفَّذ: `zakat.html` + `odhiya.html` + `qibla.html` + `manasik.html`.)**

## 3) قائمة الفحص النهائية قبل الإطلاق (QA Acceptance)

كلُّ بندٍ له معيارٌ محدّد، طريقة اختبار، ومسؤول.

| # | المجال | معيار القبول | طريقة الاختبار | المسؤول |
|---|---|---|---|---|
| 1 | الأداء — التحميل الأول | لوحة التحكم كاملة < 3 ثوانٍ على 4G | Lighthouse + جهاز Android متوسط | مطوّر |
| 2 | الأداء — الأوفلاين | المصحف/المهام/القبلة/التكبير تعمل 100% بلا إنترنت | Airplane Mode + تشغيل كل وظيفة | QA |
| 3 | الخصوصية — الزائر المجهول | لا حساب، لا بيانات تُرسل، كل شيء في القاعدة المحلية | مراقبة Network Traffic (Charles/Proxy) | مطوّر + QA |
| 4 | الدقة — أوقات الصلاة | الفارق عن المعتمد ≤ دقيقة في 10 مدن | اختبارٌ يدويٌّ (مكة/الرياض/القاهرة/جاكرتا/لندن/باريس/نيويورك/كوالالمبور/إسطنبول/لاغوس) | مدير المنتج |
| 5 | الدقة — التاريخ الهجري | يوافق إعلان الهلال الرسمي مع تصحيح ±1 يوم | مقارنة بأم القرى + تصحيح يدوي | مدير المنتج |
| 6 | الحفظ — استمرارية البيانات | الورد والمهام تبقى بعد الإغلاق وإعادة الفتح | قسري Force-stop → إعادة فتح → تحقّق | QA |
| 7 | اللهب — الاستمرارية | لا ينكسر عند تفعيل وضع العذر؛ يُستعاد بعد الربط | وضع العذر + قطع إنترنت + تغيير جهاز تجريبي | QA |
| 8 | سهولة الوصول — كبار السن | الواجهة قابلة للاستخدام بخط 200% دون تداخل/قطع | أكبر حجم خط + تصفّح كل الشاشات | مصمّم |
| 9 | التشفير — البيانات الحساسة | الأدعية الشخصية لا تظهر في Network Logs ولا في السحابة | فحص بعد حفظ دعاءٍ شخصي | مطوّر |
| 10 | صفر إعلانات | لا أيّ SDK إعلاني في الحزمة النهائية | فحص `package.json` + Android Manifest + iOS Podfile | مطوّر |

> ملاحظة PWA: البنود 2/3/6/9/10 محقَّقةٌ اليوم (أوفلاين عبر SW، تخزينٌ محلّي، لا تتبّع، الدعاء الشخصي محلّيٌّ فقط). البنود 1/4/5/8 تُقاس على الجهاز الفعلي.

## 4) مواصفات الإشعارات الذكية (مرجع)

مبدأ «التقنية الهادئة»: كل إشعارٍ مبرّر، محدود، بنبرةٍ روحية — لا إزعاج، لا تكرار، لا ضغط نفسي.

**قواعد صارمة:** حدٌّ أقصى **3 إشعارات/يوم** (لا رابع)؛ **ساعات صمتٍ إجبارية** بين منتصف الليل والفجر (الاستثناء الوحيد: تنبيه قيام الليل اليدوي)؛ **ذكاءٌ سلوكي** (إن فُتح التطبيق في الساعة الماضية لا يُرسل التالي)؛ **وضع الخلوة الرقمية** (زرٌّ يوقف كل الإشعارات 1–12 ساعة لأوقات الصلاة المطوّلة والاعتكاف).

**النبرات (أمثلة):** الفجر «الفجر يناديك — ابدأ يومك بسكينة» · الورد «وردك في انتظارك — دقائق تبقى من نهارك» · الصدقة «فرصتك للبذل لم تفت بعد» · اللهب «لهيبك لم ينطفئ — أكمل ورد اليوم» (9 مساءً فقط إذا الإنجاز < 80%) · عرفة «اليوم هو عرفة — أفضل دعاءٍ منذ عام، ابدأ الآن» · تحديث المجموعة: **Badge فقط بلا نص**.

> ملاحظة PWA: الجدولة الموثوقة في الخلفية تتطلّب Push Server أو Notification Triggers API (تجريبي). منفَّذٌ اليوم: أذونات الإشعار وتفضيلات الفئات؛ الجدولة الكاملة بندٌ خادميٌّ مستقبلي.

## 5) سيناريوهات الحافة (Edge Cases) — الاستجابة لكل موقف

| الموقف | الاستجابة الصحيحة | المكوّن المسؤول | الحالة اليوم |
|---|---|---|---|
| انقطاع الإنترنت في المشاعر | كل الوظائف تعمل من القاعدة المحلية + شريط «أوفلاين» محايد | Service Layer + SQLite | ✅ منفَّذ (SW + شريط أوفلاين) |
| تفويت يوم كامل | اللهب لا يُصفَّر؛ زر «استرداد» + خطة توزيع الفائت بنبرة «استأنف من حيث توقفت» | Streak Engine + Recovery Planner | ✅ منفَّذ (بطاقة الاسترداد) |
| وقت صلاة خاطئ (الموقع) | زر ضبط الموقع يدوياً + اختيار المدينة من قائمة | Prayer Settings + Manual Override | ✅ منفَّذ |
| تغيير الهاتف | تحذير مسبق «بياناتك محلية — اربطها» قبل الفقد؛ استرجاع بعد الربط | Cloud Sync + Migration Handler | 🔶 خادمي (موثّق) |
| المرأة في أيام العذر | وضع العذر: تُستبدل مهام الصلاة بذكرٍ وصدقة، واللهب يستمر | Excuse Mode | ✅ منفَّذ |
| انضمام مجموعة بلا حساب | المعرّف المجهول كافٍ؛ نسبة الإنجاز فقط تُشارَك | Group System + Anonymous Handler | ✅ منفَّذ |
| حذف التطبيق وإعادة تثبيته | تحذير واضح قبل الحذف «بياناتك ستُحذف نهائياً» | Deletion Warning Flow | ✅ منفَّذ (صندوق الخصوصية) |
| سعر زكاة خاطئ (API معطّل) | استخدام آخر سعرٍ مُخزَّن + تنبيه «الأسعار قد لا تكون محدّثة» | Price Cache + Stale Data Warning | ✅ منفَّذ (إدخال يدوي + تنبيه) |
| التطبيق يُفتح أثناء صلاة | وضع «لا إزعاج» يكشف تلقائياً عند إغلاق الشاشة + ميزة «وضع الصلاة» | Screen Lock Detector + DND | 🔶 يتطلّب native |

## 6) نموذج بيانات التطبيق (Data Schema)

مبدأ التصميم: «الخصوصية أولاً» — أيّ حقلٍ حسّاسٍ يبقى محلياً (مشفّراً) ولا يُرفع.

**users:** `anonymousId UUID` · `cloudUserId UUID?` (ربط Supabase Auth) · `khatmaGoal ENUM(full/half/custom)` · `dailyTarget INT` · `userMode ENUM(visitor/registered)` · `createdAt` · `lastActiveAt` · `privacyConsent BOOL`.

**quran_progress:** `anonymousId FK` · `sessionDate` · `currentSurah(1–114)` · `currentAyah` · `currentPage(1–604)` · `pagesCompletedToday` · `totalPagesCompleted` · `khatmaNumber`.

**daily_tasks:** `id UUID` · `anonymousId FK` · `taskDate` · `taskType ENUM(fasting/prayer/quran/takbeer/sadaqah/dua/custom)` · `status ENUM(pending/in_progress/completed/missed/deferred)` · `completedAt?` · `notes TEXT (encrypted — لا تُرفع)`.

**takbeer_log:** `anonymousId FK` · `takbeerDate` · `dailyCount` (يُصفَّر منتصف الليل) · `totalCount` (لا يُصفَّر) · `lastFormula(1/2/3)` · `groupContribution` (للمزامنة فقط).

**streaks:** `anonymousId FK` · `currentStreak` · `longestStreak` · `lastCompletedDate` · `exemptionMode BOOL` (اللهب يستمر معه).

> مكافئ PWA اليوم: الكائن `zad_v2` في `localStorage` يحمل هذه الحقول (worship/mushaf/takbeer/streak/groups/odhiya/zakat/personalDua/prefs/accent). الترحيل إلى SQLite/Supabase موثّق في دليل الربط.

## 7) الويدجت (Home-Screen Widget) — مرجع native

خشونةٌ رقمية: المستخدم يرى مهمةً مباشرةً دون فتح التطبيق (من «التقنية الهادئة»).

**الأحجام:** صغير 2×2 (الصلاة القادمة فقط) · متوسط 4×2 (الهجري + الصلاة + شريط تقدّم الورد) · كبير 4×4 (لوحة يومية مصغّرة: عداد عرفة + الصلاة القادمة + نسبة المهام + لهب الالتزام).

**التقنية:** iOS = WidgetKit (Swift)، Timeline Entry كل 30 دقيقة، يحسب المواقيت محلياً دون إنترنت. Android = AppWidgetProvider + WorkManager، Broadcast Receiver كل 30 دقيقة. البيانات تُقرأ من `SharedPreferences`/`UserDefaults` (يكتبها التطبيق بعد كل مهمة) — لا قراءة مباشرة من SQLite. النقر = Deep Link للصفحة المناسبة (الصلاة/الورد/المهام).

> هذا بندٌ **native حصراً** — لا يمكن للـ PWA إنشاء ويدجت شاشةٍ رئيسيةٍ حقيقي؛ يُنفَّذ في إصدار React Native.

## 8) بطاقة الحصاد الرقمي (Harvest Card) — مرجع

البطاقة المُولَّدة عبر Canvas للمشاركة الاجتماعية (تسويق عضوي):
- **الأبعاد:** 1080×1920 (ستوري 9:16) و1080×1080 (منشورات). *(منفَّذ اليوم بنسبة 9:16 عبر `<canvas>` ‎720×1280‎.)*
- **الخلفية:** تدرّج ذهبي داكن `#2D1F00 → #1A1200` + زخارف نقطية خفيفة 8%.
- **المحتوى:** الشعار «زاد العشر · 1447هـ» + الاسم المستعار (لا اسم حقيقي أبداً) + 4 إحصائيات كبرى (صفحات القرآن، التكبيرات، أيام اللهب، نسبة الإنجاز) + صف أوسمة (حتى 5 + «N+») + علامة مائية «zaad-app.com».
- **التقنية:** `canvas.toBlob('image/png', 0.95)` ثم Web Share API / تنزيل. زمن التوليد المستهدف < 2 ثانية.

> ✅ منفَّذ في لوحة التحكم (زر «شارك حصادك») بكل العناصر أعلاه.
